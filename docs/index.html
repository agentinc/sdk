Open Source SDK

Build agents.
Ship to the marketplace.

The Agentinc SDK is the developer interface for the Agentinc agent marketplace. Define agents with any LLM framework, wrap them in a universal protocol, and serve them over A2A — all with a single package.

pip install agentinc-sdk

Installation

Install the SDK with pip or uv. The only required dependency is Pydantic.

shell
# Core SDK (pydantic only)
pip install agentinc-sdk

# With A2A server support (adds FastAPI + Uvicorn)
pip install agentinc-sdk[serve]

# Development
pip install agentinc-sdk[dev]
Requires Python 3.12+. The SDK has zero platform dependencies — it never imports from internal agentinc packages.

Quickstart

Create an agent, wrap it with RawAdapter, and serve it over A2A in under 10 lines.

quickstart.py
from agentinc.sdk import RawAdapter
from agentinc.sdk.serve import serve

async def my_agent(message: str) -> str:
    return f"You said: {message}"

serve(RawAdapter(my_agent), name="echo", port=8000)
shell
# Start the server
python quickstart.py

# Call it
curl -X POST http://localhost:8000 \
  -H "Content-Type: application/json" \
  -d '{"jsonrpc":"2.0","id":1,"method":"tasks/send","params":{"id":"t1","message":{"role":"user","parts":[{"type":"text","text":"hello"}]}}}'

Core Concepts

The SDK defines the contract between your agent code and the Agentinc platform.

AgentProtocol
The universal agent contract. Implement run() to process input and yield output chunks.
ToolProtocol
Expose functions as tools the agent can call. Define schema() and call().
RawAdapter
Wraps any callable as an AgentProtocol. Supports sync, async, and streaming functions.
@tool Decorator
Turn any function into a ToolProtocol with auto-generated JSON schema from type hints.
You don't need to implement AgentProtocol directly. Use RawAdapter to wrap any function — the platform handles the rest.

AgentProtocol

The universal contract every agent must satisfy. It's a runtime-checkable Protocol, not a base class — structural subtyping means you just implement the method.

Protocol AgentProtocol

Any object with a run method matching this signature satisfies the protocol.

protocol.py
@runtime_checkable
class AgentProtocol(Protocol):
    def run(self, input: AgentInput) -> AsyncIterator[AgentOutput]: ...

Implementing directly

my_agent.py
from agentinc.sdk import AgentInput, AgentOutput, AgentProtocol

class MyAgent:
    async def run(self, input: AgentInput) -> AsyncIterator[AgentOutput]:
        yield AgentOutput(content=f"Hello! You said: {input.message}", done=True)

agent = MyAgent()
assert isinstance(agent, AgentProtocol)  # passes

ToolProtocol

The contract for tools that agents can invoke. Provides schema introspection and async execution.

Protocol ToolProtocol

Any object with schema() and call() satisfies this protocol.

protocol.py
@runtime_checkable
class ToolProtocol(Protocol):
    def schema(self) -> ToolSchema: ...
    async def call(self, tool_call: ToolCall) -> str: ...
MethodReturnsDescription
schema()ToolSchemaJSON-Schema description of the tool's parameters
call(tool_call)strExecute the tool with the given arguments and return the result

Schemas

Pydantic models that form the data contract between agents, tools, and the platform.

Model AgentInput

The input passed to every agent invocation.

FieldTypeDefaultDescription
messagestrrequiredThe user's message text
historylist[Message][]Conversation history
tool_schemaslist[ToolSchema][]Available tools for this invocation
metadatadict[str, Any]{}Arbitrary metadata
Model AgentOutput

A single chunk yielded by an agent's run() method.

FieldTypeDefaultDescription
contentstr | NoneNoneText content of this chunk
tool_callslist[ToolCall][]Tool calls the agent wants to make
doneboolFalseWhether this is the final output chunk
metadatadict[str, Any]{}Arbitrary output metadata
Model Message

A single message in the conversation history.

FieldTypeDefaultDescription
role"user" | "assistant" | "tool"requiredWho sent the message
contentstrrequiredMessage text
tool_call_idstr | NoneNoneID of the tool call this message responds to
tool_callslist[ToolCall][]Tool calls in this message
Model ToolCall

A request to invoke a specific tool.

FieldTypeDefaultDescription
idstrrequiredUnique identifier for this call
namestrrequiredName of the tool to call
argumentsdict[str, Any]{}Arguments to pass to the tool
Model ToolSchema

Describes a tool's interface for LLM function-calling.

FieldTypeDescription
namestrTool name (used in ToolCall)
descriptionstrHuman-readable description
parametersdict[str, Any]JSON Schema of the tool's parameters

@tool Decorator

Turn any function into a ToolProtocol with one line. The schema is auto-generated from type hints.

Decorator @tool

Returns a ToolWrapper instance. Works with both sync and async functions.

Bare decorator

python
from agentinc.sdk import tool

@tool
async def search(query: str, limit: int = 10) -> str:
    return f"Results for {query}"

search.schema()
# ToolSchema(name='search', description='', parameters={
#   'type': 'object',
#   'properties': {'query': {'type': 'string'}, 'limit': {'type': 'integer'}},
#   'required': ['query']
# })

With arguments

python
@tool(name="add", description="Adds two numbers")
def add_numbers(a: float, b: float) -> str:
    return str(a + b)
ParameterTypeDefaultDescription
namestr | NoneNoneOverride the tool name (defaults to function name)
descriptionstr""Override the description (defaults to docstring)

Supported type mappings

Python TypeJSON Schema
str{"type": "string"}
int{"type": "integer"}
float{"type": "number"}
bool{"type": "boolean"}
list{"type": "array"}
list[str]{"type": "array", "items": {"type": "string"}}
dict{"type": "object"}

ToolWrapper

The object returned by @tool. Satisfies ToolProtocol and can be called directly.

Class ToolWrapper

Wraps any sync or async callable with a JSON schema and ToolProtocol interface.

MethodReturnsDescription
schema()ToolSchemaThe auto-generated JSON Schema
call(tool_call)strExecute via ToolCall object
__call__(**kwargs)strExecute directly with keyword arguments
 
python
from agentinc.sdk import tool, ToolCall

@tool
def greet(name: str) -> str:
    return f"Hello, {name}!"

# Via ToolCall (how the platform dispatches)
result = await greet.call(ToolCall(id="1", name="greet", arguments={"name": "World"}))

# Direct call
result = await greet(name="World")

RawAdapter

The fastest way to turn any function into an AgentProtocol. It auto-detects your function's signature and adapts accordingly.

Class RawAdapter(fn)

Accepts any callable and exposes it as AgentProtocol.run().

Supported signatures

RawAdapter inspects your function and auto-maps arguments. No agentinc imports needed in your function.

SignatureBehavior
fn(message: str) -> strSimple request/response
fn(message: str) -> AsyncIterator[str]Streaming text chunks
fn(message: str, history: list)With conversation history (as plain dicts)
fn(message: str, history: list, tools: list)With history and available tool schemas
fn(input: AgentInput) -> AgentOutputFull agentinc-aware handler
fn(input: AgentInput) -> AsyncIterator[AgentOutput]Full streaming handler
 
python
from agentinc.sdk import RawAdapter, AgentInput

# Simple function — no SDK imports needed
async def echo(message: str) -> str:
    return f"Echo: {message}"

# Streaming
async def stream(message: str):
    for word in message.split():
        yield word + " "

# With history and tools
async def chat(message: str, history: list, tools: list):
    yield f"Got {len(history)} messages and {len(tools)} tools"

# All three work the same way:
agent = RawAdapter(echo)
async for output in agent.run(AgentInput(message="hi")):
    print(output.content)

Tool call dispatch

When your function yields a dict with tool_calls, RawAdapter converts it to a proper AgentOutput:

python
async def agent_with_tools(message: str, history: list, tools: list):
    yield {
        "tool_calls": [{"id": "1", "name": "search", "arguments": {"q": message}}],
        "done": False,
    }

A2A Server

Serve any AgentProtocol as an A2A-compliant HTTP endpoint. Supports both request/response and SSE streaming. Requires the [serve] extra.

Install the serve extra first: pip install agentinc-sdk[serve]

Endpoints

POST/JSON-RPC 2.0 endpoint

JSON-RPC Methods

Example: OpenAI Agent

Wrap an OpenAI chat completion agent with tool support.

openai_agent.py
from openai import AsyncOpenAI
from agentinc.sdk import RawAdapter, tool
from agentinc.sdk.serve import serve

client = AsyncOpenAI()

@tool(description="gets the current weather")
def get_weather(city: str) -> str:
    return f"72°F and sunny in {city}"

async def openai_agent(message: str, history: list, tools: list):
    messages = [{"role": "system", "content": "You are a helpful assistant."}]
    for msg in history:
        messages.append({"role": msg["role"], "content": msg["content"]})
    messages.append({"role": "user", "content": message})

    openai_tools = [{
        "type": "function",
        "function": {"name": t["name"], "description": t["description"], "parameters": t["parameters"]}
    } for t in tools] or None

    response = await client.chat.completions.create(
        model="gpt-4o-mini", messages=messages, tools=openai_tools,
    )
    choice = response.choices[0]

    if choice.message.tool_calls:
        yield {"tool_calls": [
            {"id": tc.id, "name": tc.function.name,
             "arguments": json.loads(tc.function.arguments)}
            for tc in choice.message.tool_calls
        ], "done": False}
    else:
        yield choice.message.content

serve(RawAdapter(openai_agent), name="openai-agent", port=8000)

Example: Anthropic Agent

Wrap a Claude-powered agent with tool support and streaming.

anthropic_agent.py
from anthropic import AsyncAnthropic
from agentinc.sdk import RawAdapter, tool
from agentinc.sdk.serve import serve

client = AsyncAnthropic()

@tool(description="looks up a fact")
def lookup(topic: str) -> str:
    return f"Facts about {topic}"

async def anthropic_agent(message: str, history: list, tools: list):
    messages = [{"role": m["role"], "content": m["content"]} for m in history]
    messages.append({"role": "user", "content": message})

    anthropic_tools = [
        {"name": t["name"], "description": t["description"], "input_schema": t["parameters"]}
        for t in tools
    ] or None

    kwargs = dict(model="claude-sonnet-4-20250514", max_tokens=1024,
                  system="You are a helpful assistant.", messages=messages)
    if anthropic_tools:
        kwargs["tools"] = anthropic_tools

    response = await client.messages.create(**kwargs)

    for block in response.content:
        if block.type == "tool_use":
            yield {"tool_calls": [{"id": block.id, "name": block.name,
                     "arguments": block.input}], "done": False}
        elif block.type == "text":
            yield block.text

serve(RawAdapter(anthropic_agent), name="anthropic-agent", port=8002)

Example: LangChain Agent

Wrap a LangChain ChatOpenAI agent with bound tools.

langchain_agent.py
from langchain_core.messages import HumanMessage, AIMessage, SystemMessage
from langchain_core.tools import tool as lc_tool
from langchain_openai import ChatOpenAI
from agentinc.sdk import RawAdapter
from agentinc.sdk.serve import serve

@lc_tool
def calculator(expression: str) -> str:
    """Evaluates a math expression."""
    return str(eval(expression))

llm = ChatOpenAI(model="gpt-4o-mini").bind_tools([calculator])

async def langchain_agent(message: str, history: list):
    messages = [SystemMessage(content="You are a helpful assistant.")]
    for msg in history:
        if msg["role"] == "user":
            messages.append(HumanMessage(content=msg["content"]))
        elif msg["role"] == "assistant":
            messages.append(AIMessage(content=msg["content"]))
    messages.append(HumanMessage(content=message))

    response = await llm.ainvoke(messages)

    if response.tool_calls:
        yield {"tool_calls": [
            {"id": tc["id"], "name": tc["name"], "arguments": tc["args"]}
            for tc in response.tool_calls
        ], "done": False}
    else:
        yield response.content

serve(RawAdapter(langchain_agent), name="langchain-agent", port=8003)

Example: CrewAI Agent

Run a CrewAI crew as an Agentinc agent.

crewai_agent.py
import asyncio
from crewai import Agent, Crew, Task
from agentinc.sdk import RawAdapter
from agentinc.sdk.serve import serve

researcher = Agent(
    role="Senior Research Analyst",
    goal="Provide thorough research on any topic",
    backstory="You are an experienced research analyst.",
    verbose=False,
)

def run_crew(message: str) -> str:
    task = Task(description=message, expected_output="Clear analysis", agent=researcher)
    crew = Crew(agents=[researcher], tasks=[task], verbose=False)
    return str(crew.kickoff())

async def crewai_agent(message: str) -> str:
    return await asyncio.to_thread(run_crew, message)

serve(RawAdapter(crewai_agent), name="crewai-agent", port=8004)

Example: Agent with Tools

Full tool loop — the agent decides which tools to call, dispatches them, feeds results back to the LLM, and responds with a final answer.

agent_with_tools.py
from openai import AsyncOpenAI
from agentinc.sdk import AgentInput, RawAdapter, ToolCall, tool
from agentinc.sdk.serve import serve

client = AsyncOpenAI()

@tool(description="Gets the current weather")
def get_weather(city: str) -> str:
    return f"72°F and sunny in {city}"

@tool(description="Evaluates a math expression")
def calculator(expression: str) -> str:
    return str(eval(expression))

TOOLS = [get_weather, calculator]
TOOL_MAP = {t.schema().name: t for t in TOOLS}

async def smart_agent(message: str, history: list, tools: list):
    messages = [{"role": "user", "content": message}]
    openai_tools = [{"type": "function", "function": t} for t in tools]

    for _ in range(5):  # max tool rounds
        resp = await client.chat.completions.create(
            model="gpt-4o-mini", messages=messages, tools=openai_tools)
        choice = resp.choices[0]

        if not choice.message.tool_calls:
            yield choice.message.content
            return

        # Dispatch each tool call
        messages.append({"role": "assistant", "content": "",
            "tool_calls": [{"id": tc.id, "type": "function",
                "function": {"name": tc.function.name,
                    "arguments": tc.function.arguments}}
                for tc in choice.message.tool_calls]})

        for tc in choice.message.tool_calls:
            result = await TOOL_MAP[tc.function.name].call(
                ToolCall(id=tc.id, name=tc.function.name,
                    arguments=json.loads(tc.function.arguments)))
            messages.append({"role": "tool", "tool_call_id": tc.id, "content": result})

serve(RawAdapter(smart_agent), name="smart-agent", port=8000)
The agent loops: LLM → tool calls → dispatch via ToolWrapper.call() → feed results back → repeat until the LLM responds with text instead of tool calls.
